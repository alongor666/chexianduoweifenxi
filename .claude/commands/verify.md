# 验证命令 (/verify)

对当前代码库状态进行全面验证。

## 检查步骤

按以下顺序执行验证:

### 1. 治理检查
```bash
node scripts/check-governance.mjs
```
- 检查文档一致性
- 验证索引完整性
- 确认护栏规则遵守

### 2. 构建检查 (如适用)
- 运行项目构建命令
- 如失败则报告错误并停止

### 3. 类型检查 (如适用)
- 运行 TypeScript/类型检查器
- 报告所有错误及文件:行号

### 4. 代码质量
- 检查代码规范
- 报告警告和错误

### 5. Git 状态
- 显示未提交的更改
- 显示自上次提交后修改的文件

## 输出格式

```
验证结果: [通过/失败]

治理检查: [OK/失败]
构建:     [OK/失败/跳过]
类型:     [OK/X 错误/跳过]
代码规范: [OK/X 问题/跳过]

准备提交 PR: [是/否]
```

如有关键问题，列出问题及修复建议。

## 参数

$ARGUMENTS 可以是:
- `quick` - 仅治理检查
- `full` - 全部检查 (默认)
- `pre-commit` - 提交前检查
- `pre-pr` - PR 前完整检查

## 项目特定检查

此项目额外检查:
- [ ] 核心业务文档未被非法修改
- [ ] INDEX.md 与实际文件同步
- [ ] BACKLOG 状态机正确
- [ ] PROGRESS 有最新记录
