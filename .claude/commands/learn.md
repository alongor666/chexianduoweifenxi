# 学习命令 (/learn)

分析当前会话，提取值得保存为技能的模式。

## 触发时机

在解决非平凡问题后的任何时刻运行 `/learn`。

## 提取内容

寻找以下模式:

### 1. 错误解决模式
- 发生了什么错误?
- 根本原因是什么?
- 如何修复的?
- 对类似错误是否可复用?

### 2. 调试技术
- 非显而易见的调试步骤
- 有效的工具组合
- 诊断模式

### 3. 变通方案
- 库的怪异行为
- API 限制
- 版本特定修复

### 4. 项目特定模式
- 发现的代码库约定
- 做出的架构决策
- 集成模式

### 5. 业务逻辑洞察 (车险项目特有)
- KPI 计算规则
- 数据转换逻辑
- 报表生成模式

## 输出格式

在 `.claude/skills/learned/[pattern-name]/SKILL.md` 创建技能文件:

```markdown
# [描述性模式名称]

**提取时间:** [日期]
**上下文:** [何时适用的简要描述]

## 问题
[此模式解决什么问题 - 要具体]

## 解决方案
[模式/技术/变通方案]

## 示例
[如适用的代码示例]

## 何时使用
[触发条件 - 什么应该激活此技能]
```

## 流程

1. 审查会话中可提取的模式
2. 识别最有价值/可复用的洞察
3. 起草技能文件
4. 请求用户确认后保存
5. 保存到 `.claude/skills/learned/`

## 注意事项

- 不提取平凡修复 (拼写错误、简单语法错误)
- 不提取一次性问题 (特定 API 中断等)
- 专注于能在未来会话中节省时间的模式
- 保持技能聚焦 - 每个技能一个模式
- 业务相关模式需确保不违反护栏规则
