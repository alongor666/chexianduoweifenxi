# 自己动手体验指南

本指南提供 5 分钟内快速验证 AgentDB 学习能力的最小步骤。所有操作均可在本地或开发环境完成，无需额外依赖。

## 前置准备

- 已安装 Node.js 与 pnpm，并完成项目依赖安装。
- 本地有可运行的 AgentDB/Claude Skills 环境（例如 `pnpm dev` 已可启动）。
- 终端可直接调用提供的 CLI 或 API 示例。

## 5 分钟体验流程

1. **启动服务**：在项目根目录执行 `pnpm dev`，确认本地端口可访问。
2. **写入第一条记忆**：
   - 通过 API 调用或 CLI 执行 `agentdb remember "广州用户提交了理赔表单"`。
   - 预期结果：返回 memory_id，状态为 `stored`。
3. **读取并复述**：
   - 使用 `agentdb recall --query "广州用户提交了什么"`。
   - 确认回复能准确引用刚写入的记忆，且带出处标识。
4. **尝试技能库调用**：
   - 添加一个计算类技能，如 `calc-cskill`，并调用 `agentdb skill calc-cskill --args "1+2+3"`。
   - 确认返回 6，且日志中出现技能命中记录。
5. **验证因果链条**：
   - 连续写入两条有前因后果的事件，再使用 `agentdb causal --query "为什么提交被加急"`。
   - 观察是否能串联因果关系并给出解释。

## 逐步验证

- 每一步都记录输出截图或日志，便于回归与分享。
- 若某步失败，优先检查环境变量、数据库连接与 API 密钥。
- 全流程完成后，可继续阅读《快速确认指南》获取更丰富的验证用例。

## 亲眼观察学习效果

- 通过多轮对话或多次 API 调用，确认系统会引用历史上下文而非仅基于单轮输入。
- 修改或删除一条记忆后再次提问，确认结果随之更新，证明存储与检索链路生效。
- 在技能调用中加入自定义提示词，观察模型是否更贴合团队语气与规范。
