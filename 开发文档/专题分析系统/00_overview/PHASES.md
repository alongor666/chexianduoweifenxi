# 项目阶段规划（Project Phases）

本文件定义《专题分析系统（车险经营）》的阶段划分、目标边界与协作约束。
所有人类与 AI 协作者，在开始任何工作前，必须以本文件作为阶段判断依据。

---

## Phase 0：问题定义与目标澄清（已完成）

**目标**
- 明确项目要解决的核心问题
- 明确"不做什么"，防止目标膨胀

**典型产出**
- 对"逻辑跳"的问题定义
- 项目核心目标共识

**当前状态**
- 已完成
- 不再新增产出

---

## Phase 1：逻辑最小闭环构建（已完成）

**目标**
- 构建可审计的专题分析逻辑闭环
- 明确每个逻辑节点存在的必要性及删除后的失效点

**核心产出**
- 《专题分析逻辑最小闭环（V0）》
- 节点定义与逻辑失效点说明

**当前状态**
- 已完成
- 逻辑闭环结构不再变更

---

## Phase 2：模板冻结与防错验证（已完成）

**目标**
- 将逻辑闭环工程化为可执行模板
- 验证模板是否能有效拦截"逻辑跳"

**核心产出**
- 《专题分析统一母模板 v1.0》（Frozen）
- Gate 流转规则与车险专属护栏
- 压力测试（Pressure Tests）
- 失败用例（Failure Cases）
- 《证据链使用与引用规范 v0.1》（Draft）

**阶段约束**
- 禁止修改《专题分析统一母模板 v1.0》的结构与规则
- 新发现的问题只能通过"失败用例 / 压力测试 / 回放"方式记录

**当前状态**
- 已完成
- 模板进入冻结期

---

## Phase 3：真实专题回放与有效性验证（进行中）

**目标**
- 在真实或高度仿真的车险专题中运行 v1.0 模板
- 验证模板是否真实拦截逻辑跳
- 识别规则盲区与高频拦截点

**允许产出**
- 专题分析回放（Replays）
- 新型逻辑跳模式记录（仅记录，不修改模板）

**禁止事项**
- 禁止直接修改《专题分析统一母模板 v1.0》
- 禁止在未完成回放前提出模板升级方案

**升级触发条件**
- 至少 3 个回放显示同一 Gate 或规则存在系统性问题

**当前状态**
- 进行中

---

## Phase 4：模板迭代与工程化（未开始）

**目标**
- 基于 Phase 3 的回放证据，对模板进行升级
- 探索半自动或自动化 Gate 校验机制

**进入条件**
- Phase 3 累计完成 ≥3 个回放
- 已识别明确、可复现的问题模式

**当前状态**
- 未开始

---

## 协作总约定（强制）

1. 所有协作者在开始任何工作前，必须确认当前 Phase；
2. 不得跨 Phase 行事（例如：在 Phase 3 修改模板）；
3. 所有变更与新增产出，必须符合当前 Phase 的"允许产出"范围；
4. 若存在 Phase 判断冲突，以本文件为最终裁决依据。
